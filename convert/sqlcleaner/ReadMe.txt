【 概　要 】

  秀丸エディタV5から変換モジュールを外出しになり、マクロの
SQLCleanをDLL化したいなぁ〜とずっと思っていたので、これを機
に移植してみました。

提供ルーチン：
「SQL整形」
「ソース生成」

ロジック的にはマクロとほぼ同じ動きをします。若干、コメント
にも対応しようとがんばってもみましたが、そもそも「汚いSQLを
整形したい。」という趣旨なので、「そんな所にコメントがあっ
ても仕方ないだろう。」とあまり凝っていません。

他、整形したSQLを、プログラムへ埋め込む為のツールも同梱しま
した。最初は別モジュールにしようと思ったのですが、「SQL整形
後しか使わないだろう」ということになりました。
「SELECT COUNT(*)」→ 「strSQL = "SELECT COUNT(*)"」と行単位
で処理するだけなので、使えたら使ってやってください。

【 ｲﾝｽﾄｰﾙ 】
１．SQLCleaner.hmfとSQLCleaner.iniを同じフォルダに入れてくだ
    さい。
    ※hmfはカレントのiniファイル（のみ）を参照します。

２．秀丸エディタへの登録は、変換モジュール一般と同じです。

３．「SQL整形」は、INIファイルの[INDENT_SKIP]セクションに登録
    された関数（実際はカッコ付きの文字列）については、改行処理
    をしないようにしています。
    "COUNT(*)"とかを一々改行処理されると見づらいので。

    要素追加・削除は、各位が実施してください。
    ELEMENTへ続く数字は、シーケンシャルにお願いします。
    （0から探して、登録が無いと読み込み終了しています。）

４．「ソース生成」は、INIファイルのHISTRY関連を見ていますが、
    手動で変更する必要はありません。
    「ソース生成」が「生成」ボタン押下時に、全部更新します。

以上。

【ｱﾝｲﾝｽﾄｰﾙ】
１．秀丸エディタ殿の変換モジュールの扱いを参考にしてください。
  「その他」→「動作環境」→「編集」→「変換」からhmf削除。
  SQLCleaner.hmfとiniを削除。

※レジストリは未使用です。

以上。

【起動方法】
※秀丸エディタ殿の変換モジュールの扱いを参考にしてください。
  文字列の選択 → 「編集」 → 「変換」 → ...
